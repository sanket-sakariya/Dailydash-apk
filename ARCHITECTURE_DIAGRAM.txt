DATA FLOW: User Adds Expense
━━━━━━━━━━━━━━━━━━━━━━━━━━

1. User enters amount via custom numpad
2. Selects category, payment mode, date
3. Confirms transaction

   AddExpenseScreen
       │
       ├─► repo.insertExpense(Expense)
       │
       ▼
   Database Layer
   ├─► SQLite: INSERT with isSynced=0
   ├─► Return expense id
       │
       ├─► Update state notifier
       │
       ▼
   DashboardScreen
   └─► Rebuilds with new expense

   [Parallel - Background Sync]
   SyncService
   ├─► Detects SQLite change
   ├─► Calls triggerSync()
   ├─► pushUnsynced() → Push to Supabase
   ├─► Conflict Resolution
   ├─► pullFromRemote() → Fetch updates
   └─► Update pendingCountNotifier

STATE MANAGEMENT FLOW: Currency Change
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

User selects EUR in Settings

  SettingsScreen
       │
       ▼
  currencyNotifier.setCurrency('EUR')
       │
       ├─► Update ValueNotifier value
       ├─► Save to SharedPreferences
       └─► ProfileService.updateCurrency('EUR')
           │
           ├─► Supabase upsert
           └─► Update ProfileService.profileNotifier
                   │
                   └─► AnalyticsScreen rebuilds (ValueListenableBuilder)
                       └─► Charts update with EUR currency symbol

KEY PATTERNS USED
━━━━━━━━━━━━━━━

1. REPOSITORY PATTERN
   Screens → DataRepository → Platform Implementation

2. SINGLETON PATTERN
   • DatabaseHelper.instance
   • AuthService.instance
   • SyncService.instance
   • ProfileService.instance

3. IMMUTABLE DATA MODELS
   • Expense.copyWith({...})
   • UserProfile.copyWith({...})

4. OUTBOX PATTERN (Sync)
   • Write locally first (isSynced=0)
   • Background push + pull
   • Last-Write-Wins conflict resolution

5. VALUE NOTIFIER PATTERN
   • Global state without external library
   • ValueListenableBuilder for reactivity
   • Listeners for side effects

6. PLATFORM ABSTRACTION
   • kIsWeb check for platform-specific code
   • Single interface, multiple implementations
